## 1. Current EN System Summary

### How it works today

The project uses a **custom hand-rolled bilingual system** — NOT auto-translate, NOT Google Translate, NOT Next.js built-in i18n.

| Layer | Mechanism | File(s) |
|-------|-----------|---------|
| Locale config | `'th' \| 'en'` type, cookie/localStorage persistence | `lib/i18n-config.ts` |
| Context provider | React Context with `t(thai, english)` function | `lib/i18n.tsx` |
| UI chrome | Inline `t('ภาษาไทย', 'English')` in every component | ~30+ component files |
| Structured content | Separate EN data files with real English content | `lib/data/services-en.ts`, `articles-en.ts`, `faqs-en.ts` |
| Team/office data | Bilingual properties on same objects (`nameEn`, `roleEn`) | `lib/data/lawyers.ts`, `advisors.ts`, `team.ts`, `office.ts` |
| Routing | File-system based `app/en/` directory (no middleware) | 21 files under `app/en/` |
| Language switcher | Changes locale context in-place (does NOT navigate to `/en`) | `components/language-switcher.tsx` |

### Verdict

**The English content is real, human-written, and professionally translated** — not machine-generated. The `services-en.ts` (421 lines), `articles-en.ts` (361 lines), and `faqs-en.ts` (40 lines) contain substantive legal English with proper terminology. The inline `t()` strings in components are also hand-written English.

This is already significantly better than auto-translate. The problem is **structural inconsistencies, missing pages, and SEO gaps** — not translation quality.

## 2. Main Problems

## 4. Recommended EN Architecture

### Three Options Evaluated

**Option A: Keep current `/en/...` file-system routing (recommended)**

| Aspect | Assessment |
|--------|-----------|
| Changes needed | Minimal — fix switcher, add 2 pages, fix SEO |
| Thai impact | Zero — Thai pages untouched |
| SEO | Good — real URLs, indexable, hreflang-ready |
| Maintainability | Good — EN pages are self-contained |
| Future expansion | Easy — add new `app/en/` pages as needed |

**Option B: Switch to locale-based `/th/...` + `/en/...` routing**

| Aspect | Assessment |
|--------|-----------|
| Changes needed | Major — rewrite all routing, redirect `/` → `/th` |
| Thai impact | **High — breaks all existing Thai URLs** |
| SEO | Excellent symmetry, but migration cost is high |
| Maintainability | Clean, but requires middleware |
| Recommendation | **Reject — too disruptive for existing Thai SEO** |

**Option C: Data-driven i18n dictionary (centralized locale files)**

| Aspect | Assessment |
|--------|-----------|
| Changes needed | Massive — refactor all components to use keys |
| Thai impact | High — all components must change |
| SEO | Same as current |
| Maintainability | Best long-term, worst short-term |
| Recommendation | **Reject — over-engineering for 2 locales** |

### Recommended: Option A — Fix and complete the current system

The current `/en/...` file-system routing is already the right approach. The issues are:
1. Language switcher behavior (doesn't navigate)
2. Missing pages (privacy, terms)
3. SEO gaps (hreflang, sitemap, metadata)

**Fix the switcher to navigate:**

```
Current: switchLocale('en') → changes context, stays on /about
Needed:  switchLocale('en') → navigates to /en/about
```

The `switchLocale` function in `lib/i18n.tsx` should use `router.push(getLocalePath(pathname, newLocale))` after persisting the locale. This is a ~10-line change.

## 5. English Tone Guideline

### Voice Profile

| Attribute | Direction |
|-----------|-----------|
| Register | Professional, formal but approachable |
| Tone | Calm, confident, trustworthy — never aggressive or salesy |
| Audience | English-speaking individuals and businesses needing Thai legal help |
| Cultural context | Explain Thai legal concepts clearly for non-Thai audience |

### Do Use

- "preliminary legal consultation"
- "review the facts and documents"
- "assess possible legal options"
- "explain risks and procedures"
- "handled under the supervision of licensed lawyers"
- "legal services in Thailand"
- "civil, criminal, family, inheritance, land, and contract matters"
- "our team will review the information and contact you"
- "does not constitute formal legal advice"
- "does not establish a lawyer-client relationship"
- "based on the facts and evidence of each matter"

### Never Use

- "We guarantee success" / "guaranteed outcome"
- "Best lawyer" / "top law firm"
- "Win every case" / "100% success rate"
- "Cheapest service" / "lowest fees"
- "Immediate victory" / "quick win"
- "We fight for you" (too aggressive for Thai legal context)
- "Justice guaranteed" (overpromising)

### Style Rules

1. Use "law office" (not "law firm" — matches Thai สำนักงาน)
2. Use "lawyer" consistently (not mixing attorney/counsel/solicitor)
3. Use "Head of Office" for หัวหน้าสำนักงาน (not "Managing Partner")
4. Use "case coordinator" for ฝ่ายประสานงานคดี
5. Use "legal advisor" for ที่ปรึกษาทางกฎหมาย
6. Spell out Thai legal terms on first use: "the Central Bankruptcy Court (ศาลล้มละลายกลาง)"
7. Use Gregorian dates for English (not Buddhist Era)
8. Use "Thailand" not "the Kingdom of Thailand" (unless legal context requires it)
